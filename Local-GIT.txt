Після чергової зміни в робочих файлах, їх слід надіслати в локальний репрезеторій, який встановлено на робочій машині..
Але перед тим як попасти з робочої папки (Working folder) в локальний репрезеторій (Local Repo), вони спочатку попадають в
спеціальну зону підготовки (Stage area) де файли проходять підготовку до переміщення в локальний репрезиторій.
Для цього виконуються наступні команди:
1) для відправки файлу в Stage area потрібно набрати команду ->     git add імя.розширення
для відправки всіх файлів ->     git add .  
для відправки файлів певного типу ->    git add *.розширення

2) Щоб переглянути статус файлів, скористайтеся командою   git status


3) для відправки файлів з Stage area в Local Repo необхідно використати команду   git commit -m "Опис змін"
При цьому файли будуть зафіксовані в Local Repo.
Кожний коміт слід супроводжувати коментарями, де доцільно вказувати зроблені нові зміни
